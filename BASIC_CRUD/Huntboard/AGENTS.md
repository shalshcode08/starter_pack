# Project conventions - Huntboard (Landed)

These are fixed rules for this project. Do not change them.

## Working style
- Guide step by step. Build collaboratively and talk through decisions before acting.
- The build plan lives in build-plan.html. Whenever a plan decision changes, update build-plan.html to match before moving on.

## Tooling
- Use pnpm for all package management and scripts. Never use npm or yarn.

## Writing
- Never use em dashes or en dashes. Use a normal hyphen "-" everywhere.

## Technical decisions
- Do not give much weight to development cost. Prefer quality, simplicity, robustness, and scalability.
- This project exists to showcase engineering skill for a resume and to help land a job. Make decisions accordingly: favor choices that demonstrate strong, professional engineering.

## Code style
- No long or unnecessary comments. Use small, professional comments only when they add real value.
- Follow clean, well-structured code patterns that an experienced professional developer would use.
