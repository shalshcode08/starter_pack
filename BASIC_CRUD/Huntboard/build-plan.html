Build Plan

Huntboard

A job-application tracker that models the search as a pipeline. Built phase by phase, each phase a working slice that can be shown.

Next.js App Router
TypeScript
PostgreSQL via Prisma
Auth.js self-hosted
Tailwind
Vercel + Neon

Phases

1

Foundation

Next.js + TypeScript app, Tailwind, Prisma, connected to Neon Postgres with separate branches for development and production, deployable shell.

Ships: a live blank app
2

Data and auth

Prisma schema for User, Application, Event. Self-hosted Auth.js with email credentials and Google (no managed service like Clerk). Every query scoped strictly to the owner.

Ships: sign in, protected routes
3

Application CRUD

Create, edit, delete, list with filters. Server-side validation with Zod. Real loading, empty, and error states.

Ships: manage applications end to end
4

Detail page and timeline

A single application view with its chronological event timeline: applied, OA, interview, follow-up, outcome.

Ships: full history per application
5

Kanban board

Drag and drop across stages: Wishlist, Applied, OA, Interview, Offer, Rejected. Dragging a card updates its status.

Ships: the visual pipeline
6

Dashboard

Derived insight: applications over time, response rate, conversion through each stage.

Ships: the analytics story
7

Polish

One-click guest login seeded with demo data. Consistent states, accessibility pass, README with live link and screenshots.

Ships: recruiter-ready experience
8

Deploy

Vercel for the app, Neon for Postgres. Publicly reachable with a guest entry point.

Ships: the live URL
