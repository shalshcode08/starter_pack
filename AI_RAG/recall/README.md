# Recall

Recall is a RAG study assistant for students. Upload lecture PDFs, group them into study sets, ask questions that are answered only from your material, and generate quizzes or flashcards that trace back to the exact source passage.

Live URL: pending Vercel deployment.

## What It Does

- Email and password authentication with isolated per-user data.
- One-click guest demo seeded from a pre-embedded sample chapter.
- PDF ingestion that extracts text, chunks it by page, embeds it in small batches, and stores vectors in PostgreSQL with pgvector.
- Grounded chat that streams answers with citations to document and page.
- Low-confidence refusal when the uploaded notes do not support an answer.
- Quiz and flashcard generation, with each item linked to its source chunk.

## Architecture

Recall is built with Next.js App Router and TypeScript. Auth.js handles sessions with JWTs, bcrypt hashes passwords, and a custom guest provider creates isolated throwaway demo users.

The data model follows this ownership chain:

```text
User -> StudySet -> Document -> Chunk
```

Conversations, messages, citations, and generated questions hang off that chain. Every data access path starts from the current user id, including vector retrieval, so one user's embeddings are never searched by another user.

The ingestion pipeline is incremental:

```text
PDF upload -> text extraction -> page-aware chunks -> 20 embeddings per process call -> ready
```

This avoids long single requests and keeps deployment friendly to serverless function limits. The database stores `Chunk.embedding` as `vector(768)`, generated by `gemini-embedding-001`, and retrieval uses cosine distance through pgvector.

## Local Setup

1. Install dependencies:

```bash
pnpm install
```

2. Create `.env`:

```bash
DATABASE_URL="postgresql://..."
AUTH_SECRET="..."
GEMINI_API_KEY="..."
```

3. Run migrations and generate the Prisma client:

```bash
pnpm db:deploy
pnpm db:generate
```

4. Seed the guest demo template:

```bash
pnpm db:seed
```

5. Start the app:

```bash
pnpm dev
```

Open `http://localhost:3000`.

## Production Deployment

Deploy on Vercel Hobby with a production Neon database.

1. Set Vercel environment variables:

```bash
DATABASE_URL="postgresql://..."
AUTH_SECRET="..."
GEMINI_API_KEY="..."
```

2. Ensure the production database can run pgvector. The initial migration includes:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

3. Run production migrations:

```bash
pnpm db:deploy
```

4. Seed the demo template against production:

```bash
pnpm db:seed
```

5. Confirm Auth.js uses the deployed host correctly. On Vercel, set `AUTH_URL` only if the deployed domain is not inferred correctly.

6. Verify the live app:

- Landing page is public.
- Guest demo creates an isolated guest account in one click.
- Upload reaches `ready` through incremental processing.
- Chat answers cite document and page.
- Out-of-scope questions are refused.
- Quiz and flashcards include source references.
