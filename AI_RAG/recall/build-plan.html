Recall - Build Plan

A study assistant that turns notes and lecture PDFs into a searchable, quiz-generating knowledge base - grounded Q&A with citations, quizzes, and flashcards from your own material.

Status: Phase 4 - PDF ingestion (up next) Cost target: Rs 0 Deploy target: Vercel Hobby

Tech Stack

Framework
Next.js (App Router) + TypeScript
Database
PostgreSQL + pgvector (Neon free tier)
ORM
Prisma
Auth

Styling
Tailwind CSS
LLM
Gemini Flash-Lite (free tier)
Embeddings
Gemini embeddings

unpdf
Package manager
pnpm (only)
Deploy
Vercel Hobby

Data Model

Hierarchy: User → StudySet → Document → Chunk (with vector embedding). Conversation/Message/Citation and Question (quiz/flashcard) hang off User and StudySet, each traceable back to a source Chunk.

Data model & schema
Done

Encode the full PRD data model in Prisma and get migrations running against pgvector.

Tasks

  • Prisma schema: User, StudySet, Document, Chunk, Conversation, Message, Citation, Question
  • Chunk.embedding as a vector column (raw SQL migration extension since Prisma lacks native vector type support)
  • Indexes: ivfflat/hnsw index on embedding, FK indexes on ownership chains
  • Seed script scaffold (used later for the guest demo)
Decided: declared the column as Unsupported("vector(768)") so Prisma generates and tracks it, then hand-edited the generated migration (via --create-only) to prepend CREATE EXTENSION IF NOT EXISTS vector and append an HNSW vector_cosine_ops index. Dimensions fixed at 768 to match Gemini text-embedding-004.
Done when: prisma migrate dev succeeds and all 8 tables exist with correct relations in Neon.
2
Authentication & authorization
Done

Every request and every vector query must be scoped strictly to the owning user.

Tasks

  • Auth.js (v5) with the Credentials provider only - email + password, no OAuth
  • JWT session strategy (no adapter, no adapter tables - keeps the User model exactly as the PRD defines it)
  • Sign-up: validate input (zod), hash password with bcryptjs, create the user
  • Server-side auth() helper to get the current user and enforce ownership on every query
  • Protected route/middleware pattern for authenticated pages
Decided: email/password only (no Google). The Credentials provider requires JWT sessions, which means we skip the Prisma adapter entirely - no Account/Session/VerificationToken tables. Passwords hashed with bcryptjs.
Done when: a user can sign up and sign in with email/password, and an unauthenticated request to any data route is rejected.

4
PDF ingestion pipeline
Up next

Upload a PDF and incrementally turn it into searchable, embedded chunks without blocking on Vercel's function timeout.

Tasks

  • Upload endpoint, storing the raw file and creating a Document row with status processing
  • Text extraction via unpdf, chunking strategy (page-aware, token-bounded)
  • Incremental processing: chunk-by-chunk or batch-by-batch calls so no single request exceeds the serverless timeout
  • Embed each chunk via Gemini embeddings, store the vector
  • Status transitions: processing → ready / failed, surfaced in the UI with polling
Decision: ingestion is driven by repeated short-lived route handler invocations (not a long-running worker), since there is no paid queue/worker service in scope.
Done when: uploading a real lecture PDF reliably reaches ready with correctly paged, embedded chunks, even for large files.
5
Grounded Q&A
Not started

Answer only from the user's own material, streamed, with exact citations, and refuse when confidence is low.

Tasks

  • Vector similarity search (cosine) over Chunks, scoped to userId + studySetId
  • Similarity threshold check: below threshold → "That isn't in your notes" refusal, no LLM call needed
  • Prompt construction that forces grounding in retrieved chunks only
  • Streamed response via Gemini Flash-Lite, rendered incrementally in the UI
  • Citations resolved to document, page, and passage; stored via the Citation table
  • Conversation and Message persistence
Done when: asking a question grounded in an uploaded PDF streams back an answer with a clickable citation to the exact page, and an out-of-scope question is refused.

Done when: generating a quiz from a study set produces items that each link back to the exact chunk they came from.
